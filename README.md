# Date Tracker — Obsidian Plugin

Автоматически отслеживает дату последнего посещения каждой заметки и показывает «забытые» заметки — те, которые вы не открывали дольше указанного количества дней.

## Возможности

- **Авто-обновление `date_last_visit`** — при открытии любой Markdown-заметки в её frontmatter записывается текущая дата.
- **Поиск забытых заметок** — сканирует все заметки и выбирает случайную, которую вы не открывали дольше заданного порога.
- **Гибкая фильтрация путей** — можно исключить или, наоборот, ограничиться только определёнными директориями через регулярные выражения.
- **Публичный API** — другие плагины или скрипты могут вызывать `window.__dateTracker.getForgottenNote()`.
- **Оптимизация** — запись в frontmatter происходит только если дата изменилась; результат выбора забытой заметки кэшируется на 10 минут.

## Установка

1. Скопируйте папку плагина в `<хранилище>/.obsidian/plugins/date-tracker/`.
2. Включите плагин в настройках Obsidian (Настройки → Сторонние плагины).

## Использование

После включения плагин автоматически проставляет поле `date_last_visit` в frontmatter каждой открываемой заметки:

```yaml
---
date_last_visit: 2026-05-30
---
```

### Настройки

Вкладка настроек плагина позволяет задать:

- **Дней без посещения** — порог (в днях), после которого заметка считается забытой.
- **Интервал ротации** — каждые N минут забытая заметка сменяется на следующую по порядку — от самой забытой к недавно посещённой. 0 = случайный выбор.
- **Исключённые пути** — regex-паттерны (каждый с новой строки). Заметки, чей путь совпадает, исключаются из поиска.
- **Включённые пути** — если не пусто, учитываются **только** заметки, чей путь совпадает хотя бы с одним паттерном.
- **Сегментов пути в названии** — сколько последних сегментов пути показывать (например, 2 = «папка/заметка»).
- **Заметок для показа** — сколько забытых заметок отображать одновременно в блоке \`\`\`forgotten-notes.

### Использование в заметках

Плагин регистрирует Markdown-блок \`\`\`forgotten-notes, который можно вставить в любую заметку.
Чтобы вставить блок, выполните команду **Date Tracker: Insert forgotten notes block** из палитры команд (Ctrl+P).

Пример:

\`\`\`
\`\`\`forgotten-notes
\`\`\`
\`\`\`

Блок отображает карточку(и) забытых заметок. Количество заметок и формат имени настраиваются в настройках плагина.

### API

Плагин экспортирует публичный API в глобальную переменную `window.__dateTracker`:

```ts
interface IForgottenNoteInfo {
  path: string;
  name: string;
  lastVisit: string | null;
  daysAgo: number | null;
}

const note: IForgottenNoteInfo | null = await window.__dateTracker.getForgottenNote();

// Получить несколько заметок
const notes: IForgottenNoteInfo[] = await window.__dateTracker.getForgottenNotes(3);
```

### Структура проекта

```
src/
├── main.ts                        # Точка входа
├── plugin.ts                      # Главный класс плагина + ForgottenNotesRenderChild
```

### Архитектура

Плагин использует сервис-ориентированную архитектуру с внедрением зависимостей. Все операции ввода-вывода с Obsidian API вынесены за тонкие адаптеры (`IMetadataReader`, `IVaultReader`, `IDataPersistence`), что позволяет тестировать сервисы без реального окружения Obsidian.
